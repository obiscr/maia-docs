Maia 是一个自托管的 DAG 工作流编排与执行服务，用于长时间自动化任务：可观测、可调试、可回放。

- **持久化**：使用 SQLite 持久化状态与输出（可保留、可追溯）
- **可观测**：实时日志/事件流（SSE），并支持回放
- **隔离执行**：可选 Runner + Sandbox 容器隔离（生产推荐）
- **可组合**：每个步骤都有明确输入/输出，并可产出产物
- **可选 Agent**：辅助生成/完善工作流（可选启用）

## 控制面 vs 执行面

在 Docker 推荐部署中，Maia 会分成两个服务：

- **App（控制面）**：UI + 调度/状态机 + SQLite + 实时 SSE
- **Runner（执行面）**：启动/销毁 sandbox 容器并把 stdout/stderr 流式回传给 App

这样的架构让你可以把“编排/状态”与“执行/隔离”分开扩展与运维。

## 什么时候你会需要 Maia？

- 你的任务**会跑很久**（分钟/小时/天），需要稳定的状态保存
- 你希望对每次执行都有**完整可观测性**（实时日志、输出、输入、产物）
- 你希望失败后可以**重试/重跑/回放**，而不是“黑盒脚本”
- 你希望把自动化拆成**可组合的步骤**，并在 UI 中可视化/可调试
