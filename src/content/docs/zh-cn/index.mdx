## 核心特性

<CardGrid>
  <Card title="持久化与可追溯" icon="open-book">
    **完整记录，永久保留**
    
    使用 SQLite 持久化所有状态和执行历史，每次执行都有完整记录。失败后可重试、可重跑、可回放，避免一次性脚本的不可控。支持长期审计和合规追溯。
  </Card>
  
  <Card title="实时可观测与回放" icon="rocket">
    **实时监控，随时回放**
    
    执行过程实时输出日志与事件，通过 SSE 推送到浏览器。输入、输出、文件与产物都可查看和下载。执行完成后支持完整回放，定位问题更快，复现更容易。
  </Card>
  
  <Card title="隔离执行与安全" icon="document">
    **容器隔离，安全可靠**
    
    生产环境推荐使用 Runner + Sandbox 容器隔离执行。控制面负责调度与持久化，执行面在独立容器中运行步骤代码，提供资源限制和安全隔离，互不影响。
  </Card>
  
  <Card title="可组合的工作流图" icon="puzzle">
    **DAG 编排，清晰可维护**
    
    用可视化编辑器构建 DAG 工作流。步骤化编排让任务结构清晰，输入输出明确，依赖关系可追踪，产物可交付。适合把零散脚本演进成长期可维护的自动化系统。
  </Card>
</CardGrid>
